Collapsible widget


Collapsible is used whe you need a view-on-demand content. It consists of a clickable bar and an optionally visible contents which get visible (or hidden) upon clicking on the bar.

Collapsible may be used standalone or as a part of a collapsible-set. Normally, the collapsible widget uses the HTML input checkbox to define its expanded and collapsed states. However, it can be used with the HTML input radio to simulate only one openeded collapsible at a time if they share the same group name or even more complex behaviour.

Collapsible widget also supports font icons. 


<div data-role="collapsible" data-animated="no" data-arrow="no" data-expanded="yes" data-e-changed="onChangedHandler()">
	<label>
		<input type="checkbox" />
		<i data-role="icon" data-icon="ion-umbrella"></i>
		<span>Collapsible</span>
	</label>
	<div data-role="container">
		<h1>1. Collapsible contents</h1>
		<p>Some content is supposed to come here.</p>
		
		<h2>1.1. Collapsible contents continued</h2>
		<p>Some continued content is supposed to come here.</p>
	</div>
</div>

<div data-role="collapsible" data-corners="smooth">
	<label>

All options are accessible throught the getOption and setOptions method of the widget [topmostMarkupElement.collapsible] object. After setting any of the options for the collapsible widget, it will be unenhanced and enhanced to apply the newly set options.

Please use the theme method of the widget to read the current theme.

  • theme

    HTML-Markup notation: data-theme

    JavaScript notation: theme


    The theme name as string to be used for the widget. Possible values 'a' or 'b'. By default, 'a' stays for dark theme, while 'b' for the bright theme. If omitted, the widget will inherit the theme from its parent.

  • corners

    HTML-Markup notation: data-corners

    JavaScript notation: corners


    Possible values:

    • flat

      The corners of the widget will be rectangular.

    • smooth

      The corners of the widget will be smoothed.


    Default is: smooth


    Remarks: The value will be ignorred if the collapsible is within a collapsible-set.

  • animated

    HTML-Markup notation: data-animated

    JavaScript notation: animated


    If set to "yes", the collapsible will expand and collapse with a smooth animation. If set to "no", the contents of the collapsible will be visible or hidden in an instant way. If omitted, it will default to "yes".

  • arrow

    HTML-Markup notation: data-arrow

    JavaScript notation: arrow


    If set to "yes", the collapsible will have an arrow indicating the expanded state of the widget. If set to "no", the idication arrow will not be displayed. If omitted, it will default to "yes".

  • expanded

    HTML-Markup notation: data-expanded

    JavaScript notation: expanded


    The initial state of the collapsible widget. If set to "yes", the collapsible will expanded. If set to "no", the widget will be collapsed. If omitted, the option will default to "no".


topmostMarkupElement.collapsible.expanded(newState)

If provided with an argument it sets the new expanded state of the collapsible. Otherwise, it just returns the current expanded state of the collapsible.

Arguments:

  • [optional] newState

    A boolean value to set the collapsible expanded state.


JS: e-changed, HTML attribute: data-e-changed

Fired when collapsible changes its expanded state. Applies to topmostMarkupElement.

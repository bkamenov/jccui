JCC class methods


You can use the 'jcc' class methods anytime you wish. However, we encourage using those methods only when implementing new functionality for 'jcc'. In other cases we advise you to use other dedicated libraries (e.g. jQuery).

jcc.version()

Returns the current JCC version as string X.Y.Z (e.g. 1.2.3).


jcc.currentHistoryState()

Returns the current history state data in the form: 


{
	id: <integer> - the ID of the state. Larger ID means newer state.
	page: <string> - the ID of the page loaded. Empty string if no page is loaded.
	popups: <array> - array of strings representing the ID's of the currently loaded popups
}


jcc.historyStateCopy(historyState)

Returns the current history state data in the form: 


{
	id: <integer> - the ID of the state. Larger ID means newer state.
	page: <string> - the ID of the page loaded. Empty string if no page is loaded.
	popups: <array> - array of strings representing the ID's of the currently loaded popups
}

Arguments:

  • historyState

    The history state to be copied.

jcc.historyStateToHash(state)

Returns the hash representation of a history state as a string.

Arguments:

  • state

    The history state to be converted to a hash value.

jcc.now()

Returns a number representing the milliseconds elapsed since the UNIX epoch.


jcc.handleHistory()

Handles the hash value when the history state changes (a new hash is provided, history state is added or overwritten).


jcc.parseHash()

Returns an array of the | delimited strings of the current location.hash.


jcc.dispatchCustomEvent(el, type, params)

Dispatches a custom DOM event to a HTML-Element.

Arguments:

  • el

    A non-null reference to the HTML-Element which is going to be the the event source.

  • type

    A string representation of the event (its name e.g. mouseout, click).

  • params

    An object containing the event arguments. You can provide null or an object with following anathomy: 
    
{
	bubbles: <boolean> - true if the event bubbles.
	cancelable: <boolean> - true if the event can be cancelled.
	details: <anything> - the data which will be seen as details in all event handlers. Can be anything.
}

jcc.getViewportRect()

Returns the rect object of the viewport in the form: 


{
	left: 0
	right: <integer> - the right-most coordinate.
	top: 0
	bottom: <integer> - the bottom-most coordinate.
	width: <integer> - the viewport width in pixels.
	height: <integer> - the viewport height in pixels.
}

jcc.isVisible(el)

Tests is a HTML-element is visible accordin to its CSS visibility and display styles.

Arguments:

  • el

    The HTML-element whose visibility is to be retrieved.

Remarks:

Please note that the function might be "expensive" as it may traverse up the DOM-tree to check if the element's parents are visible.


jcc.getElementRect(el)

Returns the rect (JS object with left, right, top, bottom, width and height properties measured in pixels) of a HTML-element in viewport coordinates - including scroll offset.

Arguments:

  • el

    The HTML-element whose rect is to be retrieved.

jcc.getElementLocalRect(el)

Returns the local rect object of a HTML-element in the form: 


{
	left: 0
	right: <integer> - the right-most coordinate.
	top: 0
	bottom: <integer> - the bottom-most coordinate.
	width: <integer> - the element width in pixels.
	height: <integer> - the element height in pixels.
}

Arguments:

  • el

    The HTML-element whose rect is to be retrieved.

jcc.isVisible(el)

Checks if a DOM-element and its parents up to the DOM-tree is not hidden (styles visibility is not "hidden" or display is not "none").

Arguments:

  • el

    The DOM-element to be checked.

jcc.resizeMonitor.addElement(el)

Adds a HTML-element to resize monitoring.

Arguments:

  • el

    The HTML-element to be monitored.

jcc.resizeMonitor.removeElement(el)

Removes a HTML-element from the resize monitor.

Arguments:

  • el

    The HTML-element to be removed from the resize monitor.

jcc.repositionMonitor.addElement(el)

Adds a HTML-element to reposition monitoring.

Arguments:

  • el

    The HTML-element to be monitored.

jcc.repositionMonitor.removeElement(el)

Removes a HTML-element from the reposition monitor.

Arguments:

  • el

    The HTML-element to be removed from the reposition monitor.

jcc.getComputedStyle(el, prop)

Returns the value of a speciffic CSS-property of a HTML element.

Arguments:

  • el

    The HTML-element to be inspected.

  • prop

    The CSS-property string.

jcc.copyCss(dst, src, style)

Copies the CSS-style(s) property from a source to a destination HTML-element. Returns an associative array with the previous style(s) values of the destination element.

Arguments:

  • dst

    The destination HTML-element of the CSS - style(s).

  • src

    The source HTML-element originator of the CSS - style(s).

  • style

    A single CSS-property string or an array of CSS-property strings.

jcc.setCss(el, css)

Sets new CSS-values to a HTML-element. Returns an associative array with the previous CSS-values of the HTML-element.

Arguments:

  • el

    The HTML-element whose CSS will be modified.

  • css

    Asscociative array with CSS-properties as keys and CSS-strings as values.

jcc.getCss(el, style)

Returns an associative array with the CSS-values of a HTML-element.

Arguments:

  • el

    The HTML-element whose CSS will be retrieved.

  • style

    A single CSS-property string or an array of CSS-property strings.

jcc.preload(imgUrl)

Preloads an image.

Arguments:

  • imgUrl

    The URL of the image to preload.

jcc.prevAll(el)

Returns an array of all DOM-nodes which are on the same level but lay before a reference HTML-element.

Arguments:

  • el

    The reference DOM-node.

jcc.nextAll(el)

Returns an array of all DOM-nodes which are on the same level but lay after a reference HTML-element.

Arguments:

  • el

    The reference DOM-node.

jcc.insertBefore(newElement, targetElement)

Inserts a new DOM-element before a given HTML-element.

Arguments:

  • newElement

    The element to be inserted.

  • targetElement

    The reference element.

jcc.insertAfter(newElement, targetElement)

Inserts a new DOM-element after a given HTML-element.

Arguments:

  • newElement

    The element to be inserted.

  • targetElement

    The reference element.

jcc.detach(el)

Removes a HTML-element from the DOM-tree.

Arguments:

  • el

    The element to be removed from the DOM.

jcc.inputTypeSupported(type)

Checks is a particular HTML-input type is supported by the current browser.

Arguments:

  • type

    The type to be checked as a string (e.g. "text", "color", "date").

jcc.whatTransitionEnd()

Returns the name of the 'transitionend' event supported by the current browser or 'null' if no transition end event is supported.


jcc.whatAnimationEnd()

Returns the name of the 'animationend' event supported by the current browser or 'null' if no animation end event is supported.


jcc.addTransitionEndListener(el, callback, capture)

Adds an event handler for the transition end event to an element and returns 'true' if transition end event is supported by the current browser.

Arguments:

  • el

    The HTML-element to which the event handler is added.

  • callback

    The event handler function.

  • [optional] capture

    Boolean indicating to use capture (true) or bubbling (false) event handling. Default is 'false'.

jcc.removeTransitionEndListener(el, callback)

Removes the event handler for the transition end event from an element and returns 'true' if transition end event is supported by the current browser.

Arguments:

  • el

    The HTML-element from which the event handler is removed.

  • callback

    The event handler function to be removed.

jcc.addAnimationEndListener(el, callback, capture)

Adds an event handler for the animation end event to an element and returns 'true' if animation end event is supported by the current browser.

Arguments:

  • el

    The HTML-element to which the event handler is added.

  • callback

    The event handler function.

  • [optional] capture

    Boolean indicating to use capture (true) or bubbling (false) event handling. Default is 'false'.

jcc.removeAnimationEndListener(el, callback)

Removes the event handler for the animation end event from an element and returns 'true' if animation end event is supported by the current browser.

Arguments:

  • el

    The HTML-element from which the event handler is removed.

  • callback

    The event handler function to be removed.

jcc.elementText(el, text)

Sets text of a HTML-element. The text will be accordingly escaped by this function, so it is the safe way of setting text to a HTML-element.

Arguments:

  • el

    The HTML-element which the text will be set to.

  • text

    The unescaped text to be set.

jcc.xmlEncode(text)

Escapes text XML entities and returns the escaped string.

Arguments:

  • text

    The unescaped text to be convertedto XML encoded text.

jcc.enhance(el)

Sets visual and behavioural properties and enhancements to a specific DOM-subtree.

Arguments:

  • el

    The DOM-hierarchy to be enhanced. The top-most DOM-node which can be provided is 'document.body'.

jcc.unenhance(el)

Removes all visual and behavioural properties and enhancements from a specific DOM-subtree.

Arguments:

  • el

    The DOM-hierarchy to be unenhanced. The top-most DOM-node which can be provided is 'document.body'.

jcc.init(el)

Initializes JCC core internals. This should be the first JCC method ever called. Without doing so, the behaviour of JCC may be undetermined. Optionally, it enhances the element starting from the argument el.

Arguments:

  • [optional] el

    The DOM-hierarchy to be enhanced. The top-most DOM-node which can be provided is 'document.body'.

jcc.sortedPrimitiveArray(array, sortNow)

Utility class for management of a sorted array of primitive type elements (numbers and strings).

Arguments:

  • array

    The JavaScript array to be enhanced.

  • sortNow

    A flag if the input array should be sorted now.

jcc.sortedPrimitiveArray.findInsertPos(element)

Returns the insert index of a new value which is not in the array yet.

Arguments:

  • element

    A number or string which is not in the array yet.

jcc.sortedPrimitiveArray.findIndex(element)

Returns the index of an existing value (its first occurence) in the array. Returns -1 if the value is not found.

Arguments:

  • element

    A number or string in the array whose index is to be retrieved.

jcc.sortedPrimitiveArray.add(element)

Adds a new element to the array.

Arguments:

  • element

    A number or string to be added.

jcc.sortedPrimitiveArray.remove(element)

Removes an element (its first occurence if there are duplicate values) from the array. It returns the index where the new element has been inserted.

Arguments:

  • element

    A number or string to be removed. It returns the index removed or -1 if the value has not been found.

jcc.sortedArray(array, sortValueName, sortNow, descending)

Utility class for management of a sorted array of object type elements.

Arguments:

  • array

    The JavaScript array to be enhanced.

  • sortValueName

    The name of the array items' object member which will be used for sorting.

  • sortNow

    A flag if the input array should be sorted now.

  • descending

    A flag if the input array should be sorted in a reverse (descending) order.

jcc.sortedArray.findInsertPosition(item)

Returns the insert index of a new item which is not in the array yet.

Arguments:

  • item

    A new item which is not in the array yet.

jcc.sortedArray.findIndexBySortValue(sortValue)

Returns the index of an existing item (its first occurence) in the array according to the value of the object member used for sorting. Returns -1 if the item is not found.

Arguments:

  • sortValue

    The value to be searched within the objects' member used for sorting.

jcc.sortedArray.findIndex(item)

Returns the index of an existing item (its first occurence) in the array. Returns -1 if the item is not found.

Arguments:

  • item

    An item in the array whose index is to be retrieved.

jcc.sortedArray.findIndexCompare(item, cmpFunc)

Returns the index of an existing item (its first occurence) in the array using a compare function. Returns -1 if the item is not found.

Arguments:

  • item

    An item in the array whose index is to be retrieved.

  • cmpFunc

    A compare function taking two arguments (the two items compared). It should return true if two items are "equal".

jcc.sortedArray.add(item)

Adds a new item to the array. It returns the index where the new item has been inserted.

Arguments:

  • item

    An item to be added.

jcc.sortedArray.remove(item, cmpFunc)

Removes an item (its first occurence if there are duplicate values) from the array. It returns the index removed or -1 if the item has not been found.

Arguments:

  • item

    The item to be removed.

  • [optional] cmpFunc

    A compare function taking two arguments (the two items compared). It should return true if two items are "equal".

jcc.alert(message, title, onenddialog, unescaped)

Displays a message on the screen using a JCC-styled dialog. See the section 'JCC properties and methods -> Properties' on how to change the alert dialog theme and default texts.

Arguments:

  • message

    The text to be displayed to the user.

  • [optional] title

    The dialog title.

  • [optional] onenddialog

    A callback function with a single integer argument (result = 0) to be called after the user dismisses the dialog in either way.

  • [optional] unescaped

    Boolean value to allow HTML-code injection in 'message' and 'title' argument and so disable escaping and allow custom visualization.

jcc.confirm(message, title, onenddialog, unescaped)

Displays a confirmation message on the screen using a JCC-styled dialog. See the section 'JCC properties and methods -> Properties' on how to change the confirmation dialog theme and default texts.

Arguments:

  • message

    The text to be displayed to the user.

  • [optional] title

    The dialog title.

  • [optional] onenddialog

    A callback function with a single integer argument (result=0, if user dismisses the dialog or presses the No-button, result=1 if user presses the Yes-button) to be called after dialog has ended.

  • [optional] unescaped

    Boolean value to allow HTML-code injection in 'message' and 'title' argument and so disable escaping and allow custom visualization.

jcc.prompt(message, title, onenddialog, cancelValue, unescaped)

Displays a prompt message waiting for user input on the screen using a JCC-styled dialog. See the section 'JCC properties and methods -> Properties' on how to change the prompt dialog theme and default texts.

Arguments:

  • message

    The text to be displayed to the user.

  • [optional] title

    The dialog title.

  • [optional] onenddialog

    A callback function with a single integer argument (result="cancel", if cancelValue is omitted, when user dismisses the dialog or presses the Cancel-button, result=value_entered if user presses the Ok-button) to be called after dialog has ended.

  • [optional] cancelValue

    The value passed to onenddialog when the user hits the Cancel-button.

  • [optional] unescaped

    Boolean value to allow HTML-code injection in 'message' and 'title' argument and so disable escaping and allow custom visualization.

jcc.datetime(input, mode, onenddialog)

Displays a date and/or time picking dialog on the screen using a JCC-styled dialog. See the section 'JCC properties and methods -> Properties' on how to change the date/time picker dialog theme and default texts.

Arguments:

  • input

    The input field to hold the results of the dialog.

  • mode

    jcc.options.datetime.DIALOG_MODE.DATE or jcc.options.datetime.DIALOG_MODE.TIME or (jcc.options.datetime.DIALOG_MODE.DATE | jcc.options.datetime.DIALOG_MODE.TIME).

  • [optional] onenddialog

    A callback function with a single integer argument (result=0, if user dismisses the dialog or presses the Cancel-button, result=date_time_value if user presses the Ok-button) to be called after dialog has ended.

jcc.colorpicker(input, mode, onenddialog)

Displays a color picking dialog on the screen using a JCC-styled dialog. See the section 'JCC properties and methods -> Properties' on how to change the color picker dialog theme and default texts.

Arguments:

  • input

    The input field to hold the results of the dialog.

  • mode

    jcc.options.colorpicker.COLOR_FORMAT.HEX or jcc.options.colorpicker.COLOR_FORMAT.RGB or jcc.options.colorpicker.COLOR_FORMAT.RGBA

  • [optional] onenddialog

    A callback function with a single integer argument (result=0, if user dismisses the dialog or presses the Cancel-button, result=color_value if user presses the Ok-button) to be called after dialog has ended.

jcc.widget(el)

The base class for all widgets is 'jcc.widget'. Its constructor takes a HTML-element as a single argument - the element to be enhanced or eventually unenhanced.

Arguments:

  • el

    The HTML-element the widget relates to.

jcc.widget.theme()

Returns the current widget theme as a string.


jcc.widget.setOptions(opt)

Sets options for a widget and refreshes the widget. NOTE: Widgets may override this method so no refresh will be called for specific options.

Arguments:

  • opt

    An associative array of options. Keys of the array should be using the naming convention used in CSS:

    'option' => 'option' as a key
    'another-option' => 'anotherOption' as a key

jcc.widget.getOption(opt)

Get the value of a particular widget option.

Arguments:

  • opt

    The option string to be retrieved. It must be in a CSS notation:

    'option' => 'option'
    'another-option' => 'anotherOption'

jcc.widget.element()

Returns the HTML-element of a widget.


jcc.widget.refresh()

Reloads the widget enhancements.


jcc.widget.destroy()

Removes all widget enhancements.

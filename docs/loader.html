Loader widget


The loader widget serves as a process indicator - indicating that something in the application is happening and requires time to finish (e.g. asynchronous request). The loader widget can be put in any element of the HTML-tree.

The loader widget can be modal (prevent the user of interacting with the elements below the loader) in an explicit (the content below will be darkened) or an implicit manner (the elements below will be just interactive)

The user may define as many loaders as needed within a single page. 


<div data-role="loader" data-isotope="spinner" data-modal="implicit" data-anchor="center center"
data-offset="0 0" data-animation="pop" data-visible="yes"
data-text="Please wait...">
</div>

All options are accessible throught the getOption and setOptions method of the widget [topmostMarkupElement.loader] object. After setting any of the options for the loader widget, it will be unenhanced and enhanced to apply the newly set options.

Please use the theme method of the widget to read the current theme.

  • theme

    HTML-Markup notation: data-theme

    JavaScript notation: theme


    The theme name as string to be used for the widget. Possible values 'a' or 'b'. By default, 'a' stays for dark theme, while 'b' for the bright theme. If omitted, the widget will inherit the theme from its parent.

  • isotope

    HTML-Markup notation: data-isotope

    JavaScript notation: isotope


    Possible values:

    • spinner

      Continuously rotating circles.

    • screw

      Pulse-rotating circles.

    • bars

      Pulsating bars.

    Default is: spinner

  • modal

    HTML-Markup notation: data-modal

    JavaScript notation: modal


    Possible values:

    • no

      Spinner is not modal - the user can interact with the controls below the loader.

    • implicit

      The items below the loader are visible but not interactive while the loader is visible.

    • explicit

      The items below the loader are darkened to show explicitly that they are not interactive while the loader is visible.

    Default is: implicit

  • anchor

    HTML-Markup notation: data-anchor

    JavaScript notation: anchor


    Specifies location within the container HTML-node where the loader should appear.

    It accepts values for the x and y axises:

    "left top", "center top", "right top",
    "left center", "center center", "right center",
    "left bottom", "center bottom", "right bottom".

    Default is: center center

  • offset

    HTML-Markup notation: data-offset

    JavaScript notation: offset


    Specifies the horizontal and vertical offset from current location in pixels for the loader widget. Negative values are allowed

    Default is: 0 0

  • animation

    HTML-Markup notation: data-animation

    JavaScript notation: animation


    The animation to be used for the loader appear and disappear. Possible values are: "none", "fade", "pop", "slideleft", "slideright", "slidefade", "slideup", "slidedown", "leftmenu", "rightmenu", "upmenu", "downmenu", "flow", "flip" and "turn".

    Default is: none

  • visible

    HTML-Markup notation: data-visible

    JavaScript notation: visible


    The visibility state of the loader widget. If yes, the loader is visible.

    Default is: no

  • text

    HTML-Markup notation: data-text

    JavaScript notation: text


    The text label to be displayed while the loader widget is visible e.g. loading, please wait.

    Default is: empty string


topmostMarkupElement.loader.visible(visibility)

Returns true if the loader is currently visible and the method is called without arguments.

Arguments:

  • [optional] visibility

    If provided, the method will show or hide the loader using the animation set.

topmostMarkupElement.loader.label(text)

Returns the text displayed while the loader is visible if the method is called without arguments.

Arguments:

  • [optional] text

    If provided, the method will set the text to be shown while the loader is visible.

e-show

Fired on the loader when it gets visible.


e-hide

Fired on the loader when it gets hidden.

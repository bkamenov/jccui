Textarea widget


A JCC styled version of the original HTML-multiline text input element known as textarea.

It supports icons for both sides. For more information, see the icon widget reference. 


<!-- Basic textarea -->
<div data-role="textarea" style="height:100px;">
	<textarea placeholder="Enter text here"></textarea>
</div>
<!-- Textarea with icon -->
<div data-role="textarea" style="height:100px;">
	<textarea placeholder="Enter text here"></textarea>
	<i data-role="icon" data-icon="ion-umbrella" data-position="left"></i>
</div>

All options are accessible throught the getOption and setOptions method of the widget [topmostMarkupElement.textarea] object. After setting any of the options for the textarea widget, it will be unenhanced and enhanced to apply the newly set options.

Please use the theme method of the widget to read the current theme.

  • theme

    HTML-Markup notation: data-theme

    JavaScript notation: theme


    The theme name as string to be used for the widget. Possible values 'a' or 'b'. By default, 'a' stays for dark theme, while 'b' for the bright theme. If omitted, the widget will inherit the theme from its parent.

  • corners

    HTML-Markup notation: data-corners

    JavaScript notation: corners


    Possible values:

    • flat

      The corners of the widget will be rectangular.

    • smooth

      The corners of the widget will be smoothed.


    Default is: smooth


topmostMarkupElement.textarea.disabled(disabled)

Returns/sets the disabled state of the textarea.

Arguments:

  • [optional] disabled

    If omitted, the method will return the disabled state of the textarea widget. If true is provided, the textarea will be set as disabled.

topmostMarkupElement.textarea.value(val)

Returns/sets the value of the textarea.

Arguments:

  • [optional] val

    If omitted, the method will return the value entered in the textarea widget. If provided, the textarea will be set with the new value provided.

The textarea widget supports all HTML-events supported by the textarea tag and all event handlers should be applied to the textarea element.

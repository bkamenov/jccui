Radio


This is the JCC replacement of the default radio input control.

The JCC radio can be used within the fieldset widget as well. For more information, look the fieldset reference. 


<label data-role="radio">
	<input type="radio" name="r" checked onclick="yourClickHandler()" />
	<span>Option 1</span>
</label>

<label data-role="radio">
	<input type="radio" name="r" />
	<span>Option 2</span>
</label>

<label data-role="radio">
	<input type="radio" name="r" disabled />
	<span>Option 3 disabled</span>
</label>

All options are accessible throught the getOption and setOptions method of the widget [topmostMarkupElement.radio]. After setting any of the options for the radio widget, it will be unenhanced and enhanced to apply the newly set options.

Please use the theme method of the widget to read the current theme.

  • theme

    HTML-Markup notation: data-theme

    JavaScript notation: theme


    The theme name as string to be used for the widget. Possible values 'a' or 'b'. By default, 'a' stays for dark theme, while 'b' for the bright theme. If omitted, the widget will inherit the theme from its parent.


topmostMarkupElement.radio.checked(newState)

If provided with an argument it sets the new checked state of the radio. Otherwise, it just returns the current checked state of the radio.

Arguments:

  • [optional] newState

    A boolean value to set the radio checked state.

topmostMarkupElement.radio.disabled(newState)

If provided with an argument it sets the new disabled state of the radio. Otherwise, it just returns the current disabled state of the radio.

Arguments:

  • [optional] newState

    A boolean value to set the radio disabled state.

Radio widget supports all the default HTML events. The handlers should be applied to the input HTML-element of the markup.

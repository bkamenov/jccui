Scrollable widget


The scrollable widget is a placeholder for large scrollable (in any direction) contents. It has custom auto-fading scroll indicators as well as kinetics scrolling (depending on the strength of dragging the mouse pointer or finger). 


<div data-role="scrollable" data-scroll-direction="all" data-kinetics="no" data-h-scrollbar-location="bottom" 
data-v-scrollbar-location="right" data-scrollbars-visibility="always"
style="border:1px solid #95a5a6; width:300px; height:200px;">
	<!-- Don't forget to define this DIV, as it is used to put the scroll bars in it. -->
	<div>
		<!-- Below this line is our scrollable content -->
		<div id="THIS IS THE CONTENT DIV" data-role="container" style="width:600px;">
			Some very long text to make the content scrollable...
		</div>
	</div>
</div>

All options are accessible throught the getOption and setOptions method of the widget [topmostMarkupElement.scrollable] object. After setting any of the options for the scrollable widget, it will be unenhanced and enhanced to apply the newly set options.

Please use the theme method of the widget to read the current theme.

  • theme

    HTML-Markup notation: data-theme

    JavaScript notation: theme


    The theme name as string to be used for the widget. Possible values 'a' or 'b'. By default, 'a' stays for dark theme, while 'b' for the bright theme. If omitted, the widget will inherit the theme from its parent.

  • kinetics

    HTML-Markup notation: data-kinetics

    JavaScript notation: kinetics


    If set to yes, the scrollable widget will still scroll deceleratingly after the mouse move or finger touch has eneded.

    Default is: yes

  • mouseWheel

    HTML-Markup notation: data-mouse-wheel

    JavaScript notation: mouseWheel


    If set to yes, the scrollable content can be scrolled up and down using the mouse wheel.

    Default is: yes

  • scrollDirection

    HTML-Markup notation: data-scroll-direction

    JavaScript notation: scrollDirection


    Possible values are:

    • horizontal

      Only horizontal scrolling will be allowed.

    • vertical

      Only vertical scrolling will be allowed.

    • all

      Scrolling in all directions will be allowed.

    Default is: vertical

  • hScrollbarLocation

    HTML-Markup notation: data-h-scrollbar-location

    JavaScript notation: hScrollbarLocation


    Possible values are:

    • left

      The horizontal scroll bar will be on the left side of the widget.

    • right

      The horizontal scroll bar will be on the right side of the widget.

    • none

      The horizontal scroll bar will not be visible.

    Default is: right

    This option makes sense only if horizontal scrolling is allowed.

  • vScrollbarLocation

    HTML-Markup notation: data-v-scrollbar-location

    JavaScript notation: vScrollbarLocation


    Possible values are:

    • top

      The vertical scroll bar will be on the top side of the widget.

    • bottom

      The vertical scroll bar will be on the bottom side of the widget.

    • none

      The vertical scroll bar will not be visible.

    Default is: bottom

    This option makes sense only if vertical scrolling is allowed.

  • scrollbarsVisibility

    HTML-Markup notation: data-scrollbars-visibility

    JavaScript notation: scrollbarsVisibility


    Possible values are:

    • always

      Scroll bars are always visible.

    • fade

      Scroll bars are visible while scrolling, then after a few seconds will fade out.

    • none

      Scroll bars are not visible at all.

    Default is: fade


e-scroll

Fired to the scrollable when the content has scrolled.

Container widget


Container is a placeholder for text, images, videos or other widgets. It defines the font styles and background color. It can be applied to body or div HTML-elements. The container widget defines the styles for HTML-headings (h1 to h6), paragraphs (p), labels (label) and links (a).

Container is should be used as a base class for higher-level placeholder widgets like the page or popup widgets. 


<body data-role="container" data-theme="a">
	<h1>Container example</h1>
	<p>
		Container widget is not really a widget but just a definition of a BODY or DIV to have a theme based font and
		background colors and padding. It also includes definition for headings (H1 to H6), paragraphs (P), labels (LABEL) and links (A).
		Another important feature of container is that the cursor is set to default, selection is disabled within its elements
		except inputs (INPUT and TEXTAREA). Browser enhancements for the elements within a container are also disabled.
	</p>

	<p>
		All widgets should be placed within a container but this is NOT a must - widgets outside a container
		must have set EXPLICITLY the styles disabling the selection and browser enhancements.
	</p>
</body>

All options are accessible throught the getOption and setOptions method of the widget [topmostMarkupElement.container] object. After setting any of the options for the container widget, it will be unenhanced and enhanced to apply the newly set options.

Please use the theme method of the widget to read the current theme.

  • theme

    HTML-Markup notation: data-theme

    JavaScript notation: theme


    The theme name as string to be used for the widget. Possible values 'a' or 'b'. By default, 'a' stays for dark theme, while 'b' for the bright theme. If omitted, the widget will inherit the theme from its parent.

  • selection

    HTML-Markup notation: data-selection

    JavaScript notation: selection


    If set to "yes" the selection will be enabled.

    Default is: no.

Pageview widget


Pageview widget is the way to simulate pages within a single HTML page. The idea is to achieve the same behaviour as if the pages within the view were real HTML pages i.e. to react on back, forward and refresh browser buttons and to be accessed through the URL but to be within the same HTML-page context e.g. too keep the single page application concept.

In addition to the multi-page simulation, the pages transition can be animated and the direction of the animation depends on the page navigation - if you follow a link or press the forward button, the page will appear using the currently set animation. Howerver, if the page is loaded via the back button, the reversed animation will be loaded.

You are allowed to have only one single pageview per HTML page. However, it may contain unlimited number of pages.

This documentation will cover the page widget as an integral part ot the pageview widget. 


<div id="pageview" data-role="pageview" data-history="yes" data-page="1st" data-animation="slideleft">
		
	<div data-role="page" id="1st" style="text-align:center;">
		<h1>PAGE 1</h1>
		<p>Put whatever widgets you want on the page.</p>
		<a href="#2nd">Go to 2nd page</a>
	</div>
	
	<div data-role="page" id="2nd" style="text-align:center;">
		<h1>PAGE 2</h1>
		<p>Put whatever widgets you want on the page.</p>
	</div>
	
</div>

All options are accessible throught the getOption and setOptions method of the widgets [topmostMarkupElement.pageview] and [pageMarkupElement.page] objects. After setting any of the options (except animation) for the pageview or page widget, it will be unenhanced and enhanced to apply the newly set option.

Please use the theme method of the widget to read the current theme.

  • [pageview] page

    HTML-Markup notation: data-page

    JavaScript notation: page


    The id of the page to be automatically loaded upon initializing the pageview widget.

    Default is: empty string

  • [pageview] history

    HTML-Markup notation: data-history

    JavaScript notation: history


    If set to "yes", the current page will be encoded in the URI hash-tag.

    If set to "no", the history browser buttons will not play any role. However, a page can be still loaded by a link or URL. In that case, each time a page is loaded, it will use the forward animation.

    A page may be loaded via URI or link.

    Default is: yes

  • [pageview] animation

    HTML-Markup notation: data-animation

    JavaScript notation: animation


    The animation to be used for the popup appear and disappear. Possible values are: "none", "fade", "pop", "slideleft", "slideright", "slidefade", "slideup", "slidedown", "leftmenu", "rightmenu", "upmenu", "downmenu", "flow", "flip" and "turn".

    Default is: none

  • [pageview and page] theme

    HTML-Markup notation: data-theme

    JavaScript notation: theme


    The theme name as string to be used for the widget. Possible values 'a' or 'b'. By default, 'a' stays for dark theme, while 'b' for the bright theme. If omitted, the widget will inherit the theme from its parent.


topmostMarkupElement.pageview.page(pageID, overrideHistoryState)

Returns the currently loaded page if called with no arguments. Alternatively, it sets the page to be loaded, if a page ID is specified.

Arguments:

  • [optional] pageID

    The ID of the page to be loaded.

  • [optional] overrideHistoryState

    If true the current history state will be overriden e.g. once the page to be loaded (C) is fully loaded and the user presses the back button, the page (A) before the previous one (B) will be loaded - as if B was never loaded. If omitted or set to false, the page to be loaded (C) will generate a new history state, e.g. when back is pressed the browser will return to the currently loaded page (B).

pageMarkupElement.page.pageview()

Returns the pageview widget set as a parent of the page.


pageMarkupElement.page.load(overrideHistoryState)

Sets the page as current one in the pageview.

Arguments:

  • [optional] overrideHistoryState

    If true the current history state will be overriden e.g. once the page to be loaded (C) is fully loaded and the user presses the back button, the page (A) before the previous one (B) will be loaded - as if B was never loaded. If omitted or set to false, the page to be loaded (C) will generate a new history state, e.g. when back is pressed the browser will return to the currently loaded page (B).

pageMarkupElement.page.isLoaded()

Returns true if the page is currently loaded in the pageview.


[pageview] e-changed

Fired to the topmostMarkupElement of the pageview when a page has been loaded.

Event members:

  • page

    The ID of the newly loaded page.

[page] e-load

Fired to the pageMarkupElement of a page widget when the page has been loaded.


[page] e-unload

Fired to the pageMarkupElement of a page widget when the page has been unloaded.

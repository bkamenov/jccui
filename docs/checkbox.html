Checkbox


This is the JCC replacement of the default checkbox input control.

The JCC checkbox can be used within the fieldset widget as well. For more information, look the fieldset reference. 


<label data-role="checkbox">
	<input type="checkbox" checked onclick="yourClickHandler()"/>
	<span>Checked</span>
</label>

<label data-role="checkbox">
	<input type="checkbox" />
	<span>Unchecked</span>
</label>

<label data-role="checkbox">
	<input type="checkbox" checked disabled />
	<span>Checked disabled</span>
</label>

<label data-role="checkbox">
	<input type="checkbox" disabled />
	<span>Unchecked disabled</span>
</label>

All options are accessible throught the getOption and setOptions method of the widget [topmostMarkupElement.checkbox]. After setting any of the options for the checkbox widget, it will be unenhanced and enhanced to apply the newly set options.

Please use the theme method of the widget to read the current theme.

  • theme

    HTML-Markup notation: data-theme

    JavaScript notation: theme


    The theme name as string to be used for the widget. Possible values 'a' or 'b'. By default, 'a' stays for dark theme, while 'b' for the bright theme. If omitted, the widget will inherit the theme from its parent.


topmostMarkupElement.checkbox.checked(newState)

If provided with an argument it sets the new checked state of the checkbox. Otherwise, it just returns the current checked state of the checkbox.

Arguments:

  • [optional] newState

    A boolean value to set the checkbox checked state.

topmostMarkupElement.checkbox.disabled(newState)

If provided with an argument it sets the new disabled state of the checkbox. Otherwise, it just returns the current disabled state of the checkbox.

Arguments:

  • [optional] newState

    A boolean value to set the checkbox disabled state.

Checkbox widget supports all the default HTML events. The handlers should be applied to the input HTML-element of the markup.

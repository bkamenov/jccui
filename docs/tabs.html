Tabs widget


The tabs widget is an implementation of tabbed pages and tabs. Page transitions are animated and the direction of the animation depends on the prder of the tabs associated - if a page is changed from a tab on the right to a page with tab on the left, the reversed animation will be played. Otherwise, if from left to right, the forward animation will be used. The active tab is highlighted. If many tab are defined and they exceed the width of the widget, they will become scrollable.

In addition, tabs can be used for page navigation (see the page widget documentation for more info) as well.

This documentation will cover the tabs, tablist, tab, tabview and tab-page widgets because together they form the whole tabs widget's features list. 


<!-- A common tabs widget -->
<div data-role="tabs" data-corners="smooth" style="width:300px; height:250px;">

	<!-- The tablist where the tabs will be placed -->
	<div id="tablist" data-role="tablist" data-active-tab="tab1" data-position="top">
		<div>
		
			<label id="tab1" data-role="tab" data-target="tabPage1" data-isotope="normal" data-mark="underscore">
				<input type="radio" name="taba"/>
				<i data-role="icon" data-icon="ion-umbrella" data-position="left"></i>
				<span>First tab</span>
			</label>
			
			<label id="tab2" data-role="tab" data-target="tabPage2" data-isotope="normal" data-mark="underscore">
				<input type="radio" name="taba"/>
				<i data-role="icon" data-icon="ion-umbrella" data-position="left"></i>
				<span>Second tab</span>
			</label>
			
		</div>
	</div>
	
	<!-- The tab view where the pages will be placed -->
	<div data-role="tabview" data-animation="slideleft">
	
		<div id="tabPage1" data-role="tab-page">
			<h1>First page</h1>
			<p>
				Some content is supposed to come here.
			</p>
		</div>
		
		<div id="tabPage2" data-role="tab-page">
			<h1>Second page</h1>
			<p>
				Some content is supposed to come here.
			</p>
		</div>
	
	</div>
	
</div>

All options are accessible throught the getOption and setOptions method of the widget [tabsMarkupElement.tabs], [tablistMarkupElement.tablist], [tabMarkupElement.tab], [tabviewMarkupElement.tabview] or [tabpageMarkupElement.tabPage] objects. After setting any of the options of the tabs, tablst, tab, tabview or tab-page widget, it will be unenhanced and enhanced to apply the newly set options.

Please use the theme method of the widget to read the current theme.

  • [tabs, tablist, tab, tabview, tabPage] theme

    HTML-Markup notation: data-theme

    JavaScript notation: theme


    The theme name as string to be used for the widget. Possible values 'a' or 'b'. By default, 'a' stays for dark theme, while 'b' for the bright theme. If omitted, the widget will inherit the theme from its parent.

  • [tabs] corners

    HTML-Markup notation: data-corners

    JavaScript notation: corners


    Possible values:

    • flat

      The corners of the widget will be rectangular.

    • smooth

      The corners of the widget will be smoothed.


    Default is: smooth

  • [tablist] activeTab

    HTML-Markup notation: data-active-tab

    JavaScript notation: activeTab


    The ID of the tab which should be active upon widget initialization. Default is empty string.

  • [tablist] position

    HTML-Markup notation: data-position

    JavaScript notation: position


    Possible values:

    • top

      The tabs will be placed above the tab view.

    • bottom

      The tabs will be placed below the tab view.


    Default is: top

  • [tablist] centerTabs

    HTML-Markup notation: data-center-tabs

    JavaScript notation: centerTabs


    If set to yes, the tabs will be horizontally centered above/below (depending on the position option) the tab view.

  • [tab] target

    HTML-Markup notation: data-target

    JavaScript notation: target


    The ID of the tab-page or page which should be displayed upon tab click. This option should be provided and be a non-empty string.

  • [tab] isotope

    HTML-Markup notation: data-position

    JavaScript notation: position


    Possible values:

    • normal

      A tab with text and eventually an icon on the left or right of the text.

    • subscript

      A tab with an icon on the center and small subscript text below.


    Default is: normal

  • [tab] mark

    HTML-Markup notation: data-mark

    JavaScript notation: mark


    Possible values:

    • underscore

      If active, the tab will be underlined.

    • content

      If active, the tab content will be highlighted and underlined.


    Default is: underscore

  • [tabview] animation

    HTML-Markup notation: data-animation

    JavaScript notation: animation


    The animation to be used for the active tab-page to appear and disappear. Possible values are: "none", "fade", "pop", "slideleft", "slideright", "slidefade", "slideup", "slidedown", "leftmenu", "rightmenu", "upmenu", "downmenu", "flow", "flip" and "turn".

    Default is "none".


tablistMarkupElement.tablist.activeTab(tab_id)

Sets/Gets the active tab for the current tab list. If the method is called without arguments, the method will return the currently set active tab.

Arguments:

  • [optional] tab_id

    The ID of the tab to be set as active.

tabMarkupElement.tab.target()

Returns the ID of the tab-page or page the tab is pointing to.


tabpageMarkupElement.tabPage.isLoaded()

Returns true if the tab-page is active and loaded.


[tablist] e-changed

Fired to the tablist when the active tab has changed.

Event members:

  • tab

    The ID of the tab being set as active.

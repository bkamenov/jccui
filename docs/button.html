Button widget


The button widget is a super set of button widgets. Depending on the 'isotope' used, you will have the button style you need.

Button widgets support font icons using the <i> tag. Icons can be placed on either side of a button - left or right. 


<!-- A button markup using HTML 'LABEL' tag -->
<label data-role="button" data-isotope="desired_button_isotope">
	<button onclick="yourClickHandler()"></button>
	<!-- Optional icon -->
	<i data-role="icon" data-icon="ion-umbrella" data-position="left"></i>
	<!-- Optional button text -->
	<span>Desired button text</span>
</label>

<!-- A button markup using HTML 'A' tag -->
<a href="#btn-click-hash" data-role="button" data-isotope="desired_button_isotope">
	<button></button>
	<!-- Optional icon -->
	<i data-role="icon" data-icon="ion-umbrella" data-position="right"></i>
	<!-- Optional button text -->
	<span>Desired button text</span>
</a>

All options are accessible throught the getOption and setOptions method of the widget [topmostMarkupElement.button] object. After setting any of the options for the button widget, it will be unenhanced and enhanced to apply the newly set options.

Please use the theme method of the widget to read the current theme.

  • theme

    HTML-Markup notation: data-theme

    JavaScript notation: theme


    The theme name as string to be used for the button. Possible values 'a' or 'b'. By default, 'a' stays for dark theme, while 'b' for the bright theme. If omitted, the button will inherit the theme from its parent.

  • corners

    HTML-Markup notation: data-corners

    JavaScript notation: corners


    The widget corners style. Could be:

    • flat

      Completely flat rect corners.

    • smooth

      Slightly rounded smooth corners.

    • round

      Left and right endges are completely round.

    Default is: "smooth".

  • action

    HTML-Markup notation: data-action

    JavaScript notation: action


    Specifies whether a button has 'special' functionality in your project. It affects ONLY the background and/or text color of your button. The value of "yes" marks the button as an action button. The value of "no" means that the button is not an 'action' button.

    Default is: "no".

  • background-color

    HTML-Markup notation: data-background-color

    JavaScript notation: backgroundColor


    Specifies a custom background color for the button. Values are any CSS color definitions (e.g. red, #ff000, rgb(255,0,0)).

  • sized

    HTML-Markup notation: data-sized

    JavaScript notation: sized


    Specifies whether a button has been sized by its CSS-style. The value of "yes" means that the button width is explicitly determined by its CSS style. The value of "no" means that the button width should be computed automatically.

    Default is: "no".

  • isotope

    HTML-Markup notation: data-theme

    JavaScript notation: theme


    A button subclass' name. This option will define which type of a button will be created. Below is a list of supported button isotopes:

    • normal

      A button with solid background. If the option 'isotope' is omitted, it defaults to 'normal'. Button background color depends on the 'action' option. Buttons of this isotope must have text and may optionally have an icon which can be placed left or right.

    • contrast

      A button with black-whitish style. The button's background color does NOT depend on the 'action' option. Buttons of this isotope must have text and may optionally have an icon which can be placed left or right.

    • quiet

      A button with transparent background and solid bright text with a color depending on the 'action' option. Buttons of this isotope must have text and may optionally have an icon which can be placed left or right and be animated.

    • contrast-quiet

      A button with transparent background and solid text in dark or bright color depending on the current theme. Buttons of this isotope must have text and may optionally have an icon which can be placed left or right and be animated.

    • light

      A button with transparent background and light text and light border with a color depending on the 'action' option. Buttons of this isotope must have text and may optionally have an icon which can be placed left or right and be animated.

    • contrast-light

      A button with transparent background, light text and light border in bright or dark color depending on current theme. Buttons of this isotope must have text and may optionally have an icon which can be placed left or right and be animated.

    • outline

      A button with transparent background and bold text and strong border with a color depending on the 'action' option. Buttons of this isotope must have text and may optionally have an icon which can be placed left or right and be animated.

    • contrast-outline

      A button with transparent background, bold text and strong border a bright or dark color depending on the current theme. Buttons of this isotope must have text and may optionally have an icon which can be placed left or right and be animated.

    • icon

      Same as 'normal' but buttons of this isotope don't have text and icon does NOT have a 'position' option.

    • contrast-icon

      Same as 'contrast' but buttons of this isotope don't have text and icon does NOT have a 'position' option.

    • icon-outline

      Same as 'outline' but buttons of this isotope don't have text and icon does NOT have a 'position' option.

    • contrast-icon-outline

      Same as 'contrast-outline' but buttons of this isotope don't have text and icon does NOT have a 'position' option.

    • icon-quiet

      Same as 'quiet' but buttons of this isotope don't have text and icon does NOT have a 'position' option. Icon is occupying the whole area of the button.

    • contrast-icon-quiet

      Same as 'contrast-quiet' but buttons of this isotope don't have text and icon does NOT have a 'position' option. Icon is occupying the whole area of the button.

    • icon-subscript

      Similar to 'quiet' but the text of these buttons is placed bellow the icon. The icon does NOT have a 'position' option.

    • contrast-icon-subscript

      Similar to 'contrast-quiet' but the text of these buttons is placed bellow the icon. The icon does NOT have a 'position' option.

    Default is: normal.


topmostMarkupElement.button.disabled(newState)

If provided with an argument it sets the new disabled state of the button. Otherwise, it just returns the current disabled state of the button.

Arguments:

  • [optional] newState

    A boolean value to set the buttin disabled state.

Remarks:

Setting the disabled state for a button using the HTML A-tag will have only visual impact on the button as the HTML link cannot be suppressed of being invoked.


Button widget supports all the default HTML events. The handlers should be applied to the button HTML-element of the markup.

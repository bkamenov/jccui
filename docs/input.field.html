Input field widget


A JCC styled version of the original HTML-input element known as input.

It supports icons for both sides. For more information, see the icon widget reference. 


<!-- Basic input field -->
<div data-role="inputfield">
	<input type="text" placeholder="Enter text here" />
</div>
<!-- Input field with icon -->
<div data-role="inputfield" data-corners="round">
	<!-- Attribute type can be text, number, color, date, time, datetime-local or datetime -->
	<input type="text" placeholder="Enter text here"/>
	<i data-role="icon" data-icon="ion-umbrella" data-position="left"></i>
</div>

All options are accessible throught the getOption and setOptions method of the widget [topmostMarkupElement.inputfield] object. After setting any of the options for the input field widget, it will be unenhanced and enhanced to apply the newly set options.

Please use the theme method of the widget to read the current theme.

  • theme

    HTML-Markup notation: data-theme

    JavaScript notation: theme


    The theme name as string to be used for the widget. Possible values 'a' or 'b'. By default, 'a' stays for dark theme, while 'b' for the bright theme. If omitted, the widget will inherit the theme from its parent.

  • isotope

    HTML-Markup notation: data-isotope

    JavaScript notation: isotope


    Possible values:

    • normal

      A standard looking input field with a flat anf thin border.

    • underscore

      A borderless input field. Once it gains focus, an animated underscore line appears on the bottom till the whole field gets underscored.

    Default is: normal

  • corners

    HTML-Markup notation: data-corners

    JavaScript notation: corners


    Possible values:

    • flat

      The corners of the widget will be rectangular.

    • smooth

      The corners of the widget will be smoothed.

    • round

      The corners of the widget will be rounded.


    Default is: smooth


topmostMarkupElement.inputfield.disabled(disabled)

Returns/sets the disabled state of the input field.

Arguments:

  • [optional] disabled

    If omitted, the method will return the disabled state of the input field widget. If true is provided, the input field will be set as disabled.

topmostMarkupElement.inputfield.value(val)

Returns/sets the value of the input field.

Arguments:

  • [optional] val

    If omitted, the method will return the value entered in the input field widget. If provided, the input field will be set with the new value provided.

The input field widget supports all HTML-events supported by the input tag and all event handlers should be applied to the input element.

The input field widget does not care about the value of the type argument of the input tag. However, JCC allows you to use its polyfills for the following types (polyfills are off per default):

  • date

    If jcc.options["datetime"]["overrideDefaultInputType"] is true the JCC date picker will be used.

  • time

    If jcc.options["datetime"]["overrideDefaultInputType"] is true the JCC time picker will be used.

  • datetime, datetime-local

    If jcc.options["datetime"]["overrideDefaultInputType"] is true the JCC date & time picker will be used.

  • color

    If jcc.options["colorpicker"]["overrideDefaultInputType"] is true the JCC color picker will be used.

Popup widget


The popup widget is the 'window' system of JCC. Using it you may create resizable, movable and custom popups, windows, menus, info boxes or other helper units for your application. Popups may be referenced by hash tags. Therefore, their visibility may depend on the current history step. Popups may appear relative to the browser window coordinates, relative to the parent HTML node of the popup or relative to another HTML element upon the same DOM tree. Together with the page widget you have the two most important 'bricks' for building your responsive application UI.

The popup may not be set to be movable by itself. To make it movable you should use the dragon widget within the popup. You can set the relateTo, position, anchor, align and animation options anytime when the widget is NOT visible, so the changes will take effect upon the next call of the show() method WITHOUT refreshing the widget.

All options are accessible throught the getOption and setOptions method of the widget [topmostMarkupElement.popup] object. After setting any of the options for the popup widget, it will be unenhanced and enhanced to apply the newly set options.

Please use the theme method of the widget to read the current theme.

  • theme

    HTML-Markup notation: data-theme

    JavaScript notation: theme


    The theme name as string to be used for the widget. Possible values 'a' or 'b'. By default, 'a' stays for dark theme, while 'b' for the bright theme. If omitted, the widget will inherit the theme from its parent.

  • modal

    HTML-Markup notation: data-modal

    JavaScript notation: modal


    Possible values are:

    • no

      The popup is not modal. The user may click the controls under the popup.

    • implicit

      The popup is modal. The controls under the popup can not be clicked until the popup is closed.

    • explicit

      The popup is modal. The controls under the popup can not be clicked until the popup is closed. The contents are 'dimmed' to visually mark that they cannot be used.


    Default is: no

  • cancelable

    HTML-Markup notation: data-cancelable

    JavaScript notation: cancelable


    If set to "yes" when user clicks outside the popup it will be closed. If cancelable is "yes" the popup option modal is set to "no", the modal option will be set to "implicit".

    Default is: yes

  • shadow

    HTML-Markup notation: data-shadow

    JavaScript notation: shadow


    If set to "yes" the popup will cast a shadow.

    Default is: yes

  • corners

    HTML-Markup notation: data-corners

    JavaScript notation: corners


    Possible values:

    • flat

      The corners of the widget will be rectangular.

    • smooth

      The corners of the widget will be smoothed.

    Default is: flat

  • history

    HTML-Markup notation: data-history

    JavaScript notation: history


    If set to "yes", the popup will be encoded in the URI hash-tag. The more history popups are visible, the longer the hash-tag e.g. "#popup1|popup2|popupN". The popup may be opened via URI or via link clicked. If set to "no", the popup may be still referenced by a link or URI but it will not be stored in the URI hash-tag once it has loaded.

    Default is: no

  • animation

    HTML-Markup notation: data-animation

    JavaScript notation: animation


    The animation to be used for the popup appear and disappear. Possible values are: "none", "fade", "pop", "slideleft", "slideright", "slidefade", "slideup", "slidedown", "leftmenu", "rightmenu", "upmenu", "downmenu", "flow", "flip" and "turn".

    Default is: none

  • reversedOut

    HTML-Markup notation: data-reversed-out

    JavaScript notation: reversedOut


    If set to "yes", the reversed animation will be played when the popup gets hidden instead of its "out" animation.

    Default is: no

  • resizeGrip

    HTML-Markup notation: data-resize-grip

    JavaScript notation: resizeGrip


    Specifies the size of the resize grip for each side of the popup. Acceptable values are any spacebar delimited combinations of a side name (left, right, top or bottom) followed by : (column) and a positive numeric value or "none" if the resizing for the corresponding side should be disabled.

    Even if one of the sides has a positive resize grip, the whole popup will be considered as a resizable popup. Resizable popups may not have the arrow option set to "yes" and the align option will be ignorred. If the popup has been resized and then hidden, the next time it is displayed it will have the same size as after it has been resized - the last popup dimensions will be remembered.

    Default is: left:none right:none top:none bottom:none

  • anchor

    HTML-Markup notation: data-anchor

    JavaScript notation: anchor


    Specifies the point on the popup used for positioning it. It accepts values for the x and y axises e.g. "left center", "right top", "5 10", "left -20" and so on.

    The anchor option is used in conjunction with the position option.

    If the align option is set, the anchor option is ignorred.

    This option may be set ONLY while the popup is NOT visible but it will not be refreshed!

    Default is: center center

  • position

    HTML-Markup notation: data-position

    JavaScript notation: position


    Specifies the popup position according to the relateTo option when the align option is NOT set. Otherwise, the position option is used as an offset value. For more information, see the align option description. The acceptable value is a space delimited combination of numeric integer values and "auto" for both x- and y- axises. The value of "auto" is a hint to the algorithm to decide where to place the popup over the corresponding axis. When not used as an offset value, "auto" means "centered" for the corresponding axis.

    This option may be set ONLY while the popup is NOT visible but it will not be refreshed!

    Default is: auto auto

  • arrow

    HTML-Markup notation: data-arrow

    JavaScript notation: arrow


    Specifies whether the popup will have a pointing arrow used as an anchor for the popup. Possible values are "yes" and "no".

    Resizable popups may NOT have the arrow set to "yes".

    Default is: no

  • align

    HTML-Markup notation: data-align

    JavaScript notation: align


    A popup is positioned according to the position and relateTo options. The align option gives the popup a hint which sides are allowed to place the popup according to the related position or element. Possible values are "left", "right", "top" and "bottom". A space delimited combination of any of these values is allowed as well.

    A value of "left right", for example, would mean that the popup may be placed on the left or right side from the relate position or element. The final decision where to place the popup depends on how big is the popup and from which of the both sides there is more place to put it within the visible area. In short, this option tells the popup: be on that side (one from the specified) from the related position or element which you fit best within the visible area.

    If the relateTo is "parent" or "window", align is "", the arrow is "no", the position and anchor options are used.

    If the arrow option is set to "yes" and align is not set or "", it will default to "left right top bottom".

    When the relateTo option points to a HTML node, the position option will be used as an offset value where positive values will mean "away" from the element and negative "inside" the element respectively. The "auto" sub-values of position will be treated as 0 (zero) when used as an offset.

    When using the align option, the popup will be readjusted if either the popup itself or the relate element move, resize or scroll to ensure the best possible visibility for the popup according to the allowed sides within the align option.

    When set, align option will force the popup to ignore the anchor option.

    This option may be set ONLY while the popup is NOT visible but it will not be refreshed!

    Default is: empty string

  • relateTo

    HTML-Markup notation: data-relate-to

    JavaScript notation: relateTo


    Specifies what the position of the popup relates to. Possibe values are:

    • window

      The popup will be positioned in window coordinates. Its position will NOT depend on the page scrolling. The popup HTML-node should be a direct child of the body HTML-node.

    • parent

      The popup will be positioned in the coordinate system of its parent HTML-node. Its position depends on the scrolling of any of its parent HTML-nodes.

    • #<html_element_id>

      The popup will be positioned relatively to a another HTML-node. The parent node of the popup should be direct or indirect parent of the related HTML element - they should share the same hierarchy.


    Even if one of the sides has a positive resize grip, the whole popup will be considered as a resizable popup. Resizable popups may not have the arrow option set to "yes" and the align option will be ignorred. If the popup has been resized and then hidden, the next time it is displayed it will have the same size as after it has been resized - the last popup dimensions will be remembered.

    This option may be set ONLY while the popup is NOT visible but it will not be refreshed!

    Default is: window


topmostMarkupElement.popup.show(overrideHistoryState)

Shows the popup.

Arguments:

  • [optional] overrideHistoryState

    Valid only for popups with history option set to "yes". If set to true, the opening of the popup will not create a new history step but the current one will include the popup as well. The same operation will be performed when popup hide() method is called, as the argument is valid until the popup is closed.

topmostMarkupElement.popup.hide(result)

Closes the popup.

Arguments:

  • [optional] result

    The result value to be provided in the e-hide event detail.result field.

topmostMarkupElement.popup.isVisible()

Returns true if the popup is visible.


e-show

Fired on the popup when it gets visible.


e-hide

Fired on the popup when it gets closed.

Event members:

  • detail.result

    The result code from the popup.hide(result) method call.

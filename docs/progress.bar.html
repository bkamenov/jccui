Progress bar widget


An implementation of two types of progress bars - a finite (determined) progress bar with two progress lines (colored differently) moving independantly, and an indeterminate progress bar (indicating an unknown process completion). 


<!-- Indeterminate infinite progress bar -->
<div data-role="progressbar" data-indeterminate="yes"></div>

<!-- Two-bar progress bar -->
<div data-role="progressbar" data-value="55" data-secondary-value="87"></div>

All options are accessible throught the getOption and setOptions method of the widget [topmostMarkupElement.progressbar] object. After setting any of the options for the progress bar widget, it will be unenhanced and enhanced to apply the newly set options.

Please use the theme method of the widget to read the current theme.

  • theme

    HTML-Markup notation: data-theme

    JavaScript notation: theme


    The theme name as string to be used for the widget. Possible values 'a' or 'b'. By default, 'a' stays for dark theme, while 'b' for the bright theme. If omitted, the widget will inherit the theme from its parent.

  • corners

    HTML-Markup notation: data-corners

    JavaScript notation: corners


    Possible values:

    • flat

      The corners of the widget will be rectangular.

    • smooth

      The corners of the widget will be smoothed.


    Default is: smooth

  • indeterminate

    HTML-Markup notation: data-indeterminate

    JavaScript notation: indeterminate


    If set to yes, the progress bar will be indeterminate (the process monitored has no definend end point) instead of determinate (the monitored process has a clear end point).

    Default is: no

  • value

    HTML-Markup notation: data-value

    JavaScript notation: value


    A positive integer value from 0 (zero progess) to 100 (completed) representing the process completion (main bar).

    Default is: 0

    Valid if indeterminate is set to yes.

  • secondaryValue

    HTML-Markup notation: data-secondary-value

    JavaScript notation: secondaryValue


    A positive integer value from 0 (zero progess) to 100 (completed) representing the process completion (secondary bar).

    Default is: 0

    Valid if indeterminate is set to yes.


topmostMarkupElement.progressbar.value(val)

Gets or sets the progress value of the main bar.

Arguments:

  • [optional] val

    The new value to be set. If omitted, the function will return the current value.

topmostMarkupElement.progressbar.svalue(val)

Gets or sets the progress value of the secondary bar.

Arguments:

  • [optional] val

    The new value to be set. If omitted, the function will return the current value.

Fieldset widget


This widget is a grouping widget to combine various widgets into a more complex one. The fieldset can combine the JCC button, checkbox, radio, dropdown and inputfield. The combination is possible in horizontal and vertical directions. 


<fieldset data-role="fieldset" data-corners="smooth" data-orientation="horizontal" data-delimiter="yes">
				
	<div>
		<label data-role="checkbox">
			<input type="checkbox" checked>
			<span>Checkbox</span>
		</label>
	</div>
	
	<div>
		<div data-role="inputfield" style="width:150px;">
			<input type="text" placeholder="Enter text here" />
			<i data-role="icon" data-icon="ion-umbrella" data-position="left"></i>
		</div>
	</div>
	
	<div>
		<label data-role="button" data-action="no" data-isotope="contrast-icon">
			<button></button>
			<i data-role="icon" data-icon="ion-umbrella"></i>
		</label>
	</div>
	
</fieldset>

All options are accessible throught the getOption and setOptions method of the widget [topmostMarkupElement.fieldset] object. After setting any of the options for the fieldset widget, it will be unenhanced and enhanced to apply the newly set options.

Please use the theme method of the widget to read the current theme.

  • theme

    HTML-Markup notation: data-theme

    JavaScript notation: theme


    The theme name as string to be used for the widget. Possible values 'a' or 'b'. By default, 'a' stays for dark theme, while 'b' for the bright theme. If omitted, the widget will inherit the theme from its parent.

  • corners

    HTML-Markup notation: data-corners

    JavaScript notation: corners


    Possible values:

    • flat

      The corners of the widget will be rectangular.

    • smooth

      The corners of the widget will be smoothed.

    • round

      The corners of the widget will be rounded. NOTE: Valid only when orientation is horizontal.


    Default is: smooth

  • orientation

    HTML-Markup notation: data-orientation

    JavaScript notation: orientation


    The direction in which the sub-widgets are stacked. Could be horizontal or vertical.

    Default is: horizontal

  • delimiter

    HTML-Markup notation: data-delimiter

    JavaScript notation: delimiter


    Defines if the sub-widgets should be delimited with a border. Could be yes or no.

    Default is: yes

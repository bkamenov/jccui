Toolbar widget


A toolbar implementation with following features:

  • Various positions: top, bottom, left, right;
  • Various sizes: tiny, small, medium, large, huge;
  • Segmentation: left/top part, middle part, right/bottom part for placement of text and controls;

The toolbar may be fixed and do not move with the scrolling content. You may have more than one toolbar within your HTML elements.

This documentation will also cover the iconic-link widget used along with toolbars. 


<div data-role="toolbar" data-fixed="no" data-orientation="horizontal" data-position="top" data-size="medium">
			
	<div class="jcc-begin">
	
		<a data-role="iconic-link" href="#">
			<i data-role="icon" data-icon="ion-android-menu"></i>
		</a>
		
	</div> 
	
	<div class="jcc-center">
	
		Some title
		
	</div>
	
	<div class="jcc-end">
	
		<a data-role="iconic-link" href="#">
			<i data-role="icon" data-icon="ion-more"></i>
		</a>
		
	</div> 
	
</div>

All options are accessible throught the getOption and setOptions method of the widget [topmostMarkupElement.toolbar] object. After setting any of the options for the toolbar widget, it will be unenhanced and enhanced to apply the newly set options.

Please use the theme method of the widget to read the current theme.

  • [toolbar, iconic-link] theme

    HTML-Markup notation: data-theme

    JavaScript notation: theme


    The theme name as string to be used for the widget. Possible values 'a' or 'b'. By default, 'a' stays for dark theme, while 'b' for the bright theme. If omitted, the widget will inherit the theme from its parent.

  • [toolbar] fixed

    HTML-Markup notation: data-fixed

    JavaScript notation: fixed


    If set to yes, the toolbar will not scroll together with the contents.

    Default is: no

  • [toolbar] orientation

    HTML-Markup notation: data-orientation

    JavaScript notation: orientation


    Possible values:

    • horizontal

      The toolbar will be horizontal.

    • vertical

      The toolbar will be vertical.

    Remarks: Affects the position option.

    Default is: horizontal

  • [toolbar] position

    HTML-Markup notation: data-position

    JavaScript notation: position


    Possible values:

    • top

      The toolbar will be placed on the top. [Valid only for horizontal toolbars]

    • bottom

      The toolbar will be placed on the bottom. [Valid only for horizontal toolbars]

    • left

      The toolbar will be placed on the left. [Valid only for vertical toolbars]

    • right

      The toolbar will be placed on the right. [Valid only for vertical toolbars]

    Default is: top [for horizontal toolbars] or left [for vertical toolbars]

  • [toolbar] size

    HTML-Markup notation: data-size

    JavaScript notation: size


    Possible values:

    • tiny

      A toolbar with thickness 16 pixels.

    • small

      A toolbar with thickness 26 pixels.

    • medium

      A toolbar with thickness 34 pixels.

    • large

      A toolbar with thickness 46 pixels.

    • huge

      A toolbar with thickness 56 pixels.

    Default is: medium

Parallax widget


The widget is a detailed model implementation of the 2D parallax effect. It is not limited by the count of the layers, nor their type, size or speed.

The parallax effect can be applied on any level in the DOM-tree. For best results in terms of smoothness, please use it within a "scrollable" widget which is part of the main JCCUI.

The parallax layers are not limited to be only images, they can be anything HTML5 supports including canvas, video and even widgets or complete forms.

The prallax implementation consists of a parallax viewport and parallax layers. Thus, this documentation will cover the parallax and parallax-layer widgets together as they form the effect and cannot exclude each other. 


<div data-role="scrollable" data-scroll-direction="all" data-kinetics="yes" data-h-scrollbar-location="bottom"
data-v-scrollbar-location="right" data-scrollbars-visibility="always"
style="position:absolute; left:0; top:0; right:0; bottom:0;">

	<div>

		<div data-role="parallax" data-direction="all" data-ortho-center="311 300" data-ortho-offset="0 0" style="position:absolute; top:0; left:0; width:623px; height:600px;">
			
			<div data-role="parallax-layer" 
			data-speed="10" 
			data-position="left top" 
			data-offset="0 0"
			data-size="auto auto"
			style="background-image:url(layer0.png); width:623px; height:600px;">
			</div>

All options are accessible throught the getOption and setOptions method of the widget [topmostMarkupElement.WIDGET] object. After setting any of the options for the WIDGET widget, it will be unenhanced and enhanced to apply the newly set options.

Please use the theme method of the widget to read the current theme.

  • theme

    HTML-Markup notation: data-theme

    JavaScript notation: theme


    The theme name as string to be used for the widget. Possible values 'a' or 'b'. By default, 'a' stays for dark theme, while 'b' for the bright theme. If omitted, the widget will inherit the theme from its parent. Not used by the parallax itself, but can be eventually used by widgets placed in the parallax layers.

  • [parallax] direction

    HTML-Markup notation: data-direction

    JavaScript notation: direction


    Possible values:

    • horizontal

      The parallax will handle only the horizontal positioning of the viewport.

    • vertical

      The parallax will handle only the vertical positioning of the viewport.

    • all

      The parallax will handle the positioning of the viewport in all directions.


    Default is: vertical

  • [parallax] orthoCenter

    HTML-Markup notation: data-ortho-center

    JavaScript notation: orthoCenter


    The HTML-window (viewport) coordinates (for the x- and y-axes) at which all the layers have no position offset e.g. are located according to the pure HTML-rules provided and NOT moved by the parallax engine. Different value types can be combined e.g. "center 20%", "150 top", ect. Please note that if you resize the browser window, the ortho center will change as well if using relational values - predefined or percentage - and will NOT change if the ortho center uses absolute values. Negative values are allowed.

    Possible values for each axis (space delimited) e.g. "center center", "50% 10%", "5 100" and so on:

    • [only for the x-axis] left

      The x-axis of the ortho center is the left most part of the HTML-window.

    • [only for the x-axis] right

      The x-axis of the ortho center is the right most part of the HTML-window.

    • [only for the y-axis] top

      The y-axis of the ortho center is the top most part of the HTML-window.

    • [only for the y-axis] bottom

      The y-axis of the ortho center is the bottom most part of the HTML-window.

    • [all axes] center

      The corresponding axis of the ortho center is at the middle of the HTML-window width or height respectively.

    • [all axes] <percentage_value>%

      The corresponding axis of the ortho center is a percentage fraction of the HTML-window width or height respectively.

    • [all axes] <absolute_value>

      The corresponding axis of the ortho center is at absolute position within the HTML-window width or height respectively.


    Default is: center center

  • [parallax] orthoOffset

    HTML-Markup notation: data-ortho-offset

    JavaScript notation: orthoOffset


    An absolute value offset for the ortho center - sort of fine tunning for the ortho offset. For example, let's take orthoOffset = "center center" and orthoOffset = "5 10". This means that the final ortho center will be 5 pixels right from half the width of the HTML-window and 10 pixels down from the half height of the HTML-window. Negative values are allowed.

    Default is: 0 0

  • [parallax-layer] speed

    HTML-Markup notation: data-speed

    JavaScript notation: speed


    A percentage representation how fast the particular layer should move according to the parallax viewport. A value of 0 means that the particular layer will not move at all when the parallax viewport moves. A value of 100 means that the particular layer will move exactly the same distance as the parallax viewport moved. Values greater than 100 will force the particular layer to move faster than the parallax viewport. Negative values will force the particular layer to move in the opposite direction of the movement of the parallax viewport.

    The general rule is that if a layer is defined later in the HTML-code, it is optically closer to the observer and thus should have larger speed value.

    Default is: 0 0

  • [parallax-layer] position

    HTML-Markup notation: data-position

    JavaScript notation: position


    The HTML-window (viewport) coordinates (for the x- and y-axes) at which the particular layer is initially located. You can use the ortho center of the parallax as a reference point (zero point) when you define your parallax model so that at the given ortho center and position the layer is located without offset - as if observed perpendicularly. Different value types can be combined e.g. "center 20%", "150 top", ect. Negative values are allowed.

    Possible values for each axis (space delimited) e.g. "center center", "50% 10%", "5 100" and so on:

    • [only for the x-axis] left

      The x-axis of the position (the left edge of the layer DIV) is at the left most part of the HTML-window.

    • [only for the x-axis] right

      The x-axis of the position (the right edge of the layer DIV) is at the right most part of the HTML-window.

    • [only for the y-axis] top

      The y-axis of the position (the top edge of the layer DIV) is at the top most part of the HTML-window.

    • [only for the y-axis] bottom

      The y-axis of the position (the bottom edge of the layer DIV) is at the bottom most part of the HTML-window.

    • [all axes] center

      The center of the layer DIV is the center of the HTML-window width or height respectively/

    • [all axes] <percentage_value>%

      The position of the corresponding edge (left or top) of the layer DIV is a percentage fraction of the HTML-window width or height respectively.

    • [all axes] <absolute_value>

      The position of the corresponding edge (left or top) of the layer DIV is at absolute position within the HTML-window width or height respectively.


    Default is: center center

  • [parallax-layer] offset

    HTML-Markup notation: data-offset

    JavaScript notation: offset


    An absolute value offset for the layer position - sort of fine tunning for the layer position. For example, let's take position = "center center" and offset = "5 10". This means that the final layer position will be 5 pixels right from half the width of the HTML-window and 10 pixels down from the half height of the HTML-window. Negative values are allowed.

    Default is: 0 0

  • [parallax-layer] size

    HTML-Markup notation: data-size

    JavaScript notation: size


    Size of the corresponding layer (in terms of HTML-window coordinates). Depending on the size value specified, the layer DIV may be scaled to match the desired criteria.

    Possible values can be single or space delimited per axis. Omitted values (except contain and cover) fot the y-axis default to auto. List of the possible values:

    • [single value] contain

      The whole layer DIV is maximized as much as possible to fit the HTML-window and preserve the original aspect ratio.

    • [single] cover

      The longest side of the layer DIV is stretched to match the HTML-window width or height respectively. The aspect ratio is preserved. Thus, the other edge if the layer DIV is scaled accordingly. However, it might not be always visible depending on the original layer DIV and HTML-window sizes. This value ensures that the whole HTML-window are is covered.

    • [single or space delimited] auto

      When specified for the corresponding axis, the original size of the layer DIV (as difined in the HTML code) will be used.

    • [space delimited] <percentage_value>%

      Width or height of the layer DIV percentage fraction referred to the HTML-window width or height respectively.

    • [all axes] <absolute_value>

      Width or height of the layer DIV in pixels. Please note that the layer DIV should already have non-zero dimensions (width and height).


    Default is: 100% 100%


layerDiv.parallaxLayer.update(containerRect, windowRect, orthoCenter)

The method is exposed for overriding - if one wants to add more effects over the parallax layer like extra scaling, opacity, rotations and so on.

Arguments:

  • containerRect

    The parallax viewport rect in HTML-window coordinates.

  • windowRect

    The HTML-window rect.

  • orthoCenter

    The finally computed orthoCenter in absolute pixels for the current containerRect and windowRect.

Remarks

The layers, which are not loaded (enhanced by the JCC engine) yet, will not be affected.


e-layer-loaded

Fired by each parallax layer once it has been enhanced by the JCC engine.

Event members:

  • detail

    A reference to the HTML-node of the parallax layer.

e-loaded

Fired by the parallax once all parallax layers are loaded.
